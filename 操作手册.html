系统概述

系统简介

标书资质自动匹配系统是一个自动化标书处理平台，主要功能包括：

  • 📥 标书爬取：自动从浙江省政府采购网爬取招标公告
  • 📄 文件解析：自动解析PDF、Word等格式的标书文件
  • 🤖 AI分析：使用AI提取资质要求并比对公司资质
  • 📊 报告生成：生成详细的匹配分析报告

系统架构

用户界面 (Streamlit Web) ↓ 流程执行模块 ↓ ┌──────────┬──────────┬──────────┬──────────┐ │ 爬虫 │ 解析器 │ AI分析 │ 报告生成 │ └──────────┴──────────┴──────────┴──────────┘ ↓ 数据库 (SQLite)

快速开始

1. 启动系统

# 激活虚拟环境
.venv\Scripts\activate

# 启动Web界面
streamlit run app.py

2. 访问系统

浏览器自动打开，或手动访问：http://localhost:8501

3. 首次使用

  1. 配置公司资质：
    • 进入"资质管理"页面
    • 添加公司各类资质信息
  2. 配置A类证书：
    • 进入"A类证书管理"
    • 添加或编辑证书信息
  3. 配置B类规则：
    • 进入"B类规则管理"
    • 添加或编辑匹配规则
  4. 执行全流程：
    • 进入"流程执行"页面
    • 选择"全流程"
    • 设置爬取数量
    • 点击"开始执行"

功能模块详解

1. 项目概览

位置：首页（默认页面）

功能：

  • 显示项目统计信息（总数、已完成、推荐参与等）
  • 显示今日项目统计
  • 显示项目状态分布图表
  • 显示最近的项目列表

操作：

  • 查看项目详情：点击项目名称
  • 筛选项目：使用筛选条件
  • 导出数据：点击"导出Excel"

2. 流程执行

位置：侧边栏 → "流程执行"

功能模块：

2.1 标书爬虫

功能：从浙江省政府采购网爬取招标公告

操作步骤：

  1. 选择"标书爬虫"
  2. 设置爬取数量（1-200）
  3. 点击"开始爬取"
  4. 查看实时进度和统计信息
  5. 可随时暂停或中断

注意事项：

  • 爬取数量建议设置为 20-50（根据需求）
  • 爬取过程可能需要较长时间，请耐心等待
  • 网络不稳定时可能失败，可重试

2.2 文件解析

功能：解析已下载的标书文件

操作步骤：

  1. 选择"文件解析"
  2. 点击"开始解析"
  3. 系统自动解析所有待解析项目
  4. 查看解析结果

支持格式：

  • PDF (.pdf)
  • Word (.doc, .docx)
  • 压缩包 (.zip) - 自动解压并解析

2.3 AI资质分析

功能：使用AI提取资质要求并比对

操作步骤：

  1. 选择"AI资质分析"
  2. 选择模型类型（本地/云）
  3. 点击"开始分析"
  4. 查看分析进度和结果

分析内容：

  • 提取项目资质要求
  • 比对公司资质
  • 生成匹配结果（推荐参与/不推荐参与）

2.4 报告生成

功能：生成Excel格式的分析报告

操作步骤：

  1. 选择"报告生成"
  2. 点击"生成报告"
  3. 下载生成的Excel文件

报告内容：

  • 项目基本信息
  • 资质要求
  • 匹配结果
  • 推荐建议

2.5 全流程

功能：自动执行所有步骤

操作步骤：

  1. 选择"全流程"
  2. 设置爬取数量
  3. 点击"开始执行"
  4. 系统自动执行：爬取 → 解析 → AI分析 → 报告生成

优势：

  • 一键完成所有操作
  • 自动处理各步骤间的依赖关系
  • 实时显示进度

3. 资质管理

位置：侧边栏 → "资质管理"

3.1 公司资质管理

功能：管理公司各类资质信息

操作：

  • 添加资质：点击"添加资质"，填写信息
  • 编辑资质：点击"编辑"按钮
  • 删除资质：点击"删除"按钮
  • 导入Excel：批量导入资质信息

资质分类：

  • 企业资质
  • 人员资质
  • 设备要求
  • 业绩要求
  • 其他要求

3.2 A类证书管理

功能：管理A类证书（体系认证、资质证书等）

操作：

  • 添加证书：填写证书名称、编号、颁发机构等
  • 编辑证书：修改证书信息
  • 启用/禁用：控制证书是否参与匹配

证书类型：

  • 体系认证（ISO9001、ISO14001等）
  • 资质证书
  • 许可证
  • 检测报告
  • 其他证书

3.3 B类规则管理

功能：管理匹配规则（自动判定规则）

操作：

  • 添加规则：设置触发条件和结论
  • 编辑规则：修改规则内容
  • 启用/禁用：控制规则是否生效

规则类型：

  • 证书类
  • 业绩类
  • 检测报告类
  • 人员资质类
  • 其他类

4. 存储管理

位置：侧边栏 → "存储管理"

功能：

  • 查看磁盘使用情况
  • 查看文件统计信息
  • 手动清理文件
  • 配置自动清理规则

操作：

  • 查看统计：自动显示文件数量和大小
  • 清理文件：点击"清理文件"按钮
  • 配置规则：修改 config.py 中的 STORAGE_CONFIG

5. 任务调度

位置：侧边栏 → "任务调度"

功能：

  • 设置定时任务（Windows任务计划程序）
  • 配置自动执行时间
  • 查看任务状态

操作：

  • 创建任务：设置执行时间和参数
  • 删除任务：移除已创建的任务
  • 测试任务：立即执行一次任务

操作流程

流程一：单次全流程执行

适用场景：需要立即处理一批标书

步骤：

  1. 启动系统
  2. 进入"流程执行"页面
  3. 选择"全流程"
  4. 设置爬取数量（例如：20）
  5. 点击"开始执行"
  6. 等待执行完成
  7. 查看结果和报告

预计时间：

  • 爬取：5-10分钟（20个项目）
  • 解析：10-20分钟
  • AI分析：20-40分钟
  • 总计：35-70分钟

流程二：分步执行

适用场景：需要分步控制或排查问题

步骤：

  1. 爬取标书：
    • 选择"标书爬虫"
    • 设置数量并执行
    • 等待完成
  2. 解析文件：
    • 选择"文件解析"
    • 执行解析
    • 检查解析结果
  3. AI分析：
    • 选择"AI资质分析"
    • 选择模型类型
    • 执行分析
  4. 生成报告：
    • 选择"报告生成"
    • 生成并下载报告

流程三：定时自动执行

适用场景：每天自动处理新标书

步骤：

  1. 进入"任务调度"页面
  2. 创建定时任务：
    • 设置执行时间（例如：每天凌晨2点）
    • 设置爬取数量
    • 选择模型类型
  3. 保存任务
  4. 系统将自动执行

常见操作场景

场景一：首次使用系统

  1. 配置基础信息：
    • 添加公司资质
    • 添加A类证书
    • 配置B类规则
  2. 测试系统：
    • 设置爬取数量为 1-2 个
    • 执行全流程
    • 检查结果是否正确
  3. 正式使用：
    • 根据需求设置爬取数量
    • 执行全流程
    • 查看报告

场景二：处理解析失败的文件

  1. 查看失败项目：
    • 进入"项目概览"
    • 筛选状态为"异常"的项目
  2. 检查失败原因：
    • 查看错误信息
    • 检查文件是否存在
    • 检查文件格式是否支持
  3. 重新解析：
    • 修复文件问题（如需要）
    • 选择"文件解析"重新执行
    • 或手动处理该文件

场景三：更新公司资质

  1. 进入资质管理：
    • 点击"资质管理" → "公司资质管理"
  2. 添加或编辑资质：
    • 点击"添加资质"或"编辑"
    • 更新资质信息
    • 保存
  3. 重新分析项目（如需要）：
    • 进入"AI资质分析"
    • 重新执行分析

场景四：批量导入资质

  1. 准备Excel文件：
    • 格式：资质分类 | 资质内容
    • 保存为 .xlsx 格式
  2. 导入：
    • 进入"公司资质管理"
    • 点击"导入Excel"
    • 选择文件并上传
  3. 验证：
    • 检查导入的资质是否正确

场景五：清理旧文件

  1. 查看存储情况：
    • 进入"存储管理"
    • 查看文件统计
  2. 手动清理：
    • 点击"清理文件"
    • 选择要清理的文件类型
    • 确认清理
  3. 配置自动清理：
    • 编辑 config.py
    • 设置 STORAGE_CONFIG
    • 系统将自动清理

故障排除

问题一：爬取失败

可能原因：

  • 网络连接问题
  • 网站反爬虫机制
  • 配额已满

解决方法：

  1. 检查网络连接
  2. 等待一段时间后重试
  3. 减少爬取数量
  4. 检查日志文件

问题二：文件解析失败

可能原因：

  • 文件格式不支持
  • 文件损坏
  • Word COM组件不可用
  • 文件路径错误

解决方法：

  1. 检查文件格式（支持：PDF、Word、ZIP）
  2. 检查文件是否完整
  3. 运行 python test_file_parser.py 诊断
  4. 检查文件路径配置

问题三：AI分析失败

可能原因：

  • Ollama服务未启动
  • 模型未下载
  • 网络连接问题
  • 请求频率过高

解决方法：

  1. 检查Ollama服务：curl http://localhost:11434/api/tags
  2. 下载模型：ollama pull qwen3-coder:480b-cloud
  3. 检查网络连接
  4. 等待一段时间后重试（避免频率限制）

问题四：报告生成失败

可能原因：

  • 没有可生成报告的数据
  • 文件权限问题
  • 磁盘空间不足

解决方法：

  1. 确保有已比对的项目
  2. 检查 reports/ 目录权限
  3. 检查磁盘空间

问题五：系统运行缓慢

可能原因：

  • 爬取数量过多
  • AI分析耗时
  • 系统资源不足

解决方法：

  1. 减少爬取数量
  2. 使用云模型（更快）
  3. 升级硬件配置
  4. 关闭其他占用资源的程序

数据备份

重要文件

需要定期备份的文件：

  • tender_system.db - 数据库文件
  • config.py - 配置文件
  • reports/ - 报告目录
  • 重要的标书文件

备份方法

# 备份数据库
copy tender_system.db backup\tender_system_20251220.db

# 备份配置文件
copy config.py backup\config_20251220.py

# 备份报告
xcopy reports backup\reports_20251220\ /E /I
